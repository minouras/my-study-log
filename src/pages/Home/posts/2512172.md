# 🎯 結論：Hono インスタンス（Hono<AppType>）の主なメソッド一覧

以下を全部理解すれば **Hono の土台は完璧**です👇

## 🔵 1. **HTTP メソッド系**（ルーティング）

```ts
app.get(path, handler);
app.post(path, handler);
app.put(path, handler);
app.delete(path, handler);
app.patch(path, handler);
app.options(path, handler);
app.head(path, handler);
```

→ これは Express とほぼ同じで、純粋に「このメソッドで、このパスで、この処理する」というだけ。

---

## 🔵 2. **汎用ルート定義**

```ts
app.on(method, path, handler);
```

例：

```ts
app.on(['GET', 'POST'], '/hello', (c) => c.text('Hi'));
```

---

## 🔵 3. **ミドルウェア登録**

```ts
app.use(path?, middleware)
```

- パスを省略 → 全ルート
- パスを指定 → そのパス配下に適用

例：

```ts
app.use('*', logger());
app.use('/admin/*', authMiddleware);
```

---

## 🔵 4. **ルーターのネスト（route）**

```ts
app.route(prefix, subApp);
```

複数ファイルに分割するとき一番使う。

例：

```ts
app.route('/posts', postsRoute);
```

これで `/posts/...` に subApp のルーティングを委譲できる。

---

## 🔵 5. **全てのルートにマッチ（all）**

```ts
app.all(path, handler);
```

メソッド関係なく受けたい時。

例：

```ts
app.all('/health', (c) => c.json({ ok: true }));
```

---

## 🔵 6. **エラーハンドリング**

### エラーハンドラのセット

```ts
app.onError((err, c) => { ... })
```

### NotFound ハンドラ

```ts
app.notFound((c) => c.text('Not found', 404));
```

---

## 🔵 7. **型関連（高度だけど重要）**

### 型注入

```ts
const app = new Hono<{ Variables: {...} }>()
```

### ルートレベルで型を絞る

```ts
app.get<{ Params: { id: string } }>('/posts/:id', ...)
```

---

## 🔵 8. **その他の内部ユーティリティ**

### fetch

```ts
app.fetch;
```

Hono の “本体”とも言える関数。
`serve()` や Cloudflare Workers に渡すために存在している。

### basePath

```ts
app.basePath('/api');
```

### fire（サーバレス環境）

普段は使わないが、Lambda 等で便利。

---

# 🧩 それぞれの動作イメージと設計思想

Hono は **Fetch API ベース**で作られているので、
Express のような「req / res 直操作」ではなく、

- `c.req`（Request ラッパ）
- `c.res`（Response のためのヘルパー）

を返すシンプルな構造。

つまり：

- **メソッド** → HTTP メソッドごとの分岐
- **use** → ミドルウェア
- **route** → ルータの分割
- **fetch** → Hono アプリそのもの（Cloudflare Workers と互換）

という理解で OK。

# 📋 主要プロパティまとめ表（日本語）

| メソッド            | 役割                                    |
| ------------------- | --------------------------------------- |
| get/post/put/delete | HTTPルート登録                          |
| all                 | 全メソッド受け取り                      |
| on                  | メソッドに柔軟対応                      |
| use                 | ミドルウェア登録                        |
| route               | ルートのグルーピング                    |
| onError             | 全体の例外処理                          |
| notFound            | 404 ページ                              |
| basePath            | 全体の prefix 設定                      |
| fetch               | Hono アプリ本体（サーバー起動時に利用） |

# 📘 実務で最も重要な 4 つだけ覚えるならこれ

### ✔ app.use

ミドルウェア（ログ、認証、CORS など）

### ✔ app.route

複数ファイルに分割する時の要

### ✔ app.get/app.post

純粋な API / ルーティング

### ✔ app.onError

例外キャッチの基本
